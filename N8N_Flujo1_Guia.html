3 · Nodo 2 · Switch por evento
2
Switch Enrutar por tipo de evento Separar las 3 rutas

Agrega el nodo Switch (Core Nodes) y configura estas 3 reglas:

Salida (Output)CondiciónValor
Output 1 → Rama A {{ $json.evento }} equals lead_capturado
Output 2 → Rama B {{ $json.evento }} equals lead_descartado
Output 3 → Rama C {{ $json.evento }} equals lead_calificado
ℹ️
En N8N el nodo Switch separa el flujo en outputs. Conecta cada output al primer nodo de su rama correspondiente.

5 · Rama B · lead_descartado
🔴 Rama B — lead_descartado · No califica por filtro automático
3B
Set Calcular motivo de descarte Para Supabase y WhatsApp

Agrega un nodo Set para detectar el motivo de descarte:

Variable nueva → Name: motivoDescarte

Value (Expression):

// Pega esta expresión en el campo Value:
{{
  $json.ingresos === 'Menor a 150 mil pesos'
    ? 'Ingresos insuficientes (< 150k MXN/mes)'
    : $json.antiguedad === 'Menor a 6 meses'
    ? 'Antigüedad insuficiente (< 6 meses)'
    : $json.constitucion === 'Persona Física con Actividad Empresarial' && $json.buroPF === 'Malo'
    ? 'Buró malo — Persona Física'
    : $json.buroPMEmpresa === 'Malo'
    ? 'Buró malo — Empresa'
    : $json.buroPMAccionista === 'Malo'
    ? 'Buró malo — Accionista principal'
    : 'Motivo no determinado'
}}

8 · Activar el flujo y conectar ProfilingForm.tsx
  • ①En N8N, haz clic en el toggle "Inactive → Active" arriba a la derecha del editor del flujo.
  • ②Copia la URL de producción del nodo Webhook (pestaña "Production URL"). Tiene el formato: https://santiagor17.app.n8n.cloud/webhook/[ID]
  • ③Abre components/ProfilingForm.tsx y reemplaza la línea:

    // Antes:
const N8N_WEBHOOK_URL_PROD = '';

// Después (pega tu URL real):
const N8N_WEBHOOK_URL_PROD = 'https://santiagor17.app.n8n.cloud/webhook/[ID-REAL]';
  • ④Prueba llenando el formulario en local (npm run dev) hasta el paso 0 → verifica en N8N que llega el evento lead_capturado.
  • ⑤Completa el formulario hasta el final con datos que califiquen → verifica evento lead_calificado + mensaje WhatsApp + registro en Supabase.
  • ⑥Prueba con datos que no califiquen (ej: ingresos "Menor a 150 mil pesos") → verifica evento lead_descartado.
  • ⑦Cuando todo funcione: npm run build → subir carpeta dist/ a Hostinger public_html/
⚠️
Meta test number: Con el número de prueba de Meta, WhatsApp solo envía mensajes a los números que hayas verificado en la consola de Meta Developers (máx 5 números en modo sandbox). El número del asesor debe estar en esa lista. Cuando tengas un número oficial, solo cambia "to": "52..." en los nodos de WhatsApp.
